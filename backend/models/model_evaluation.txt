Model Evaluation (Operational Metrics)
====================================

This project records lightweight, operational evaluation metrics for the NLP
pipeline based on the current processed dataset. These are meant to answer:
"Is the model producing outputs and what is the label distribution?"

This is NOT a supervised accuracy/precision/recall evaluation because the
dataset in this repo does not include ground-truth sentiment labels.

Source Of Truth
---------------
- CSV history: data/processed/model_evaluation_history.csv
- Database table: model_evaluations (SQLite by default in data/app.db)
- API surface: GET /admin/overview (includes latest_evaluation)

How To Refresh
--------------
The backend writes new evaluation rows on startup (lifespan hook) by calling
evaluate_model_quality(). You can also trigger it manually with:

python -c "from src.core.evaluation import evaluate_model_quality; print(evaluate_model_quality())"

This file can be regenerated anytime with:

python scripts/update_model_docs.py

Interpretation
--------------
- coverage: Proxy based on processed row count (capped at 1.0).
- avg_confidence: Derived from sentiment_score distribution in processed data.
- *_ratio: Ratio of labels in processed data.
